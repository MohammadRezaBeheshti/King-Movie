# اتصال Vite به Django

## هدف

در این مرحله ابزار Vite به پروژه اضافه شد تا فرآیند توسعه و Build فایل‌های CSS و JavaScript سریع‌تر و استانداردتر انجام شود.

---

## ابزارهای مورد استفاده

- Vite
- Tailwind CSS v4
- Django Template
- Template Tag اختصاصی

---

## معماری انتخاب شده

```
frontend/
│
├── assets/
│   ├── fonts/
│   ├── images/
│   └── icons/
│
├── css/
└── js/

↓

Vite Build

↓

static/build/
```

در این معماری، تمامی فایل‌های منبع (Source) داخل پوشه `frontend` قرار دارند و Vite خروجی نهایی را در پوشه `static/build` تولید می‌کند.

---

## مدیریت فایل‌های Build

Vite در زمان Build نام فایل‌ها را به همراه Hash تولید می‌کند.

نمونه:

```
app-BCTH4lEj.js
app-mcl4YvYI.css
```

این کار باعث جلوگیری از کش شدن فایل‌های قدیمی در مرورگر می‌شود.

---

## Template Tag اختصاصی

برای جلوگیری از تغییر دستی نام فایل‌های Build، یک Template Tag اختصاصی در اپلیکیشن `common` پیاده‌سازی شد.

این Template Tag فایل `manifest.json` را خوانده و مسیر صحیح فایل‌های CSS و JavaScript را در اختیار Templateهای Django قرار می‌دهد.

مزایا:

- حذف تغییرات دستی در Templateها
- استفاده از آخرین نسخه فایل‌های Build
- استقلال از کتابخانه‌های جانبی

---

## چالش‌های پیاده‌سازی

در ابتدای توسعه، استفاده از کتابخانه `django-vite` بررسی شد، اما به دلیل اینکه پروژه از Django Template استفاده می‌کند و Frontend مستقل ندارد، این روش کنار گذاشته شد.

در نهایت یک Template Tag اختصاصی توسعه داده شد که دقیقاً متناسب با نیاز پروژه است.

---

## نتیجه

در پایان این مرحله، فرآیند مدیریت فایل‌های CSS و JavaScript به صورت خودکار انجام شده و زیرساخت Frontend پروژه برای ادامه توسعه آماده گردید.
