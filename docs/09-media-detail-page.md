# پیاده‌سازی صفحه جزئیات رسانه (Media Detail Page)

## ۱. هدف مرحله

در این مرحله صفحه جزئیات فیلم، سریال و انیمیشن پیاده‌سازی شد.

این صفحه یکی از اصلی‌ترین بخش‌های پروژه King Movie است، زیرا بیشتر قابلیت‌های آینده مانند:

- علاقه‌مندی‌ها
- لیست تماشا
- امتیازدهی
- نظرات کاربران
- نمایش فصل‌ها و قسمت‌ها

بر پایه این صفحه توسعه داده خواهند شد.

---

# ۲. مسیر صفحه

برای نمایش جزئیات هر رسانه از مسیر زیر استفاده شد:

```
/media/<slug>/
```

مثال:

```
/media/breaking-bad/
```

Slug باعث می‌شود صفحات دارای URL خوانا و مناسب موتورهای جستجو باشند.

---

# ۳. ساختار Template

برای جلوگیری از ایجاد فایل‌های بزرگ، صفحه به کامپوننت‌های کوچک‌تر تقسیم شد.

ساختار:

```
templates/

pages/
    media-detail.html


components/

media/
    detail/
        hero.html
        info.html
        metadata.html
        actions.html
        cast.html
        seasons.html
        trailer.html
        reviews.html
```

---

# ۴. معماری View

View وظیفه دریافت اطلاعات از دیتابیس را دارد.

Template فقط مسئول نمایش اطلاعات است.

برای دریافت رسانه از:

```python
get_object_or_404()
```

استفاده شد.

---

# ۵. بهینه‌سازی Queryها

برای جلوگیری از ایجاد Queryهای اضافی، از ابزارهای بهینه‌سازی Django استفاده شد.

## select_related

برای روابط ForeignKey:

مانند:

- کارگردان
- اطلاعات وابسته

استفاده شد.

---

## prefetch_related

برای روابط Many To Many:

مانند:

- ژانرها
- بازیگران
- فصل‌ها
- قسمت‌ها

استفاده شد.
