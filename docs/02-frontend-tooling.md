# راه‌اندازی Frontend پروژه

## هدف

هدف از این مرحله ایجاد زیرساخت مناسب برای توسعه رابط کاربری مدرن و قابل توسعه بود.

---

## انتخاب Tailwind CSS

برای طراحی رابط کاربری از Tailwind CSS نسخه 4 استفاده شده است.

مهم‌ترین دلایل این انتخاب عبارت‌اند از:

- سرعت بالای توسعه
- کاهش حجم CSS
- قابلیت شخصی‌سازی بالا
- مناسب بودن برای طراحی Responsive
- پشتیبانی مناسب از حالت Dark Mode

---

## انتخاب Vite

به منظور مدیریت Assetهای پروژه از Vite استفاده شد.

مزایای Vite:

- سرعت بسیار بالا
- Hot Reload
- Build سریع
- مدیریت ساده فایل‌های CSS و JavaScript

---

## ساختار Frontend

```
frontend/

css/

app.css

theme.css

fonts.css

js/

app.js
```

در این ساختار فایل‌های توسعه در پوشه Frontend قرار گرفته و خروجی نهایی توسط Vite تولید خواهد شد.

---

## طراحی سیستم استایل

در این پروژه به جای استفاده مستقیم از رنگ‌ها، از CSS Variables استفاده خواهد شد.

این روش مزایای زیر را دارد:

- تغییر آسان Theme
- پیاده‌سازی ساده Dark Mode
- مدیریت بهتر رنگ‌ها
- افزایش قابلیت نگهداری پروژه

---

## فونت پروژه

فونت اصلی پروژه Arad انتخاب شده است.

این فونت به صورت محلی در پروژه نگهداری می‌شود تا وابستگی به CDN وجود نداشته باشد.

---

## نتیجه

در پایان این مرحله زیرساخت Frontend پروژه آماده گردید تا در مراحل بعدی طراحی رابط کاربری بر پایه Tailwind CSS انجام شود.
