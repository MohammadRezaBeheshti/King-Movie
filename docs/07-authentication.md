# پیاده‌سازی سیستم احراز هویت

## ۱. هدف مرحله

در این مرحله سیستم احراز هویت کاربران پروژه پیاده‌سازی شد. این بخش یکی از مهم‌ترین قسمت‌های هر سامانه تحت وب است؛ زیرا امکاناتی مانند علاقه‌مندی‌ها، لیست تماشا، ثبت امتیاز، ارسال نظر و مدیریت پروفایل تنها برای کاربران وارد شده قابل استفاده هستند.

برای پیاده‌سازی این بخش از سیستم احراز هویت آماده Django استفاده شد و نیازی به توسعه یک سیستم اختصاصی نبود.

---

# ۲. مفاهیم مورد استفاده

## Authentication چیست؟

Authentication یا احراز هویت فرآیندی است که مشخص می‌کند کاربر واقعاً همان شخصی است که ادعا می‌کند.

در این پروژه کاربر با استفاده از:

- نام کاربری
- رمز عبور

وارد سیستم می‌شود.

---

## Authorization چیست؟

Authorization یا مجوز دسترسی مشخص می‌کند پس از ورود، کاربر به چه بخش‌هایی دسترسی داشته باشد.

در این پروژه:

- کاربران عادی فقط به امکانات سایت دسترسی دارند.
- مدیر سیستم از طریق پنل مدیریت Django اطلاعات سایت را مدیریت می‌کند.

---

## چرا از User پیش‌فرض Django استفاده شد؟

Django به صورت پیش‌فرض یک مدل کاربر بسیار کامل در اختیار برنامه‌نویس قرار می‌دهد.

مزایای استفاده از آن:

- امنیت بالا
- تست‌شده بودن
- توسعه سریع‌تر
- عدم نیاز به پیاده‌سازی مجدد سیستم ورود کاربران

به همین دلیل برای این پروژه از مدل User پیش‌فرض Django استفاده شد.

---

# ۳. مراحل پیاده‌سازی

برای پیاده‌سازی سیستم احراز هویت مراحل زیر انجام شد.

### ایجاد فرم‌های ورود و ثبت‌نام

داخل اپلیکیشن Accounts فایل زیر ایجاد شد:

```
accounts/forms.py
```

در این فایل از فرم‌های آماده Django استفاده شد:

- AuthenticationForm
- UserCreationForm

این فرم‌ها اعتبارسنجی اطلاعات کاربران را به صورت خودکار انجام می‌دهند.

---

### ایجاد Viewها

در فایل:

```
accounts/views.py
```

Viewهای زیر پیاده‌سازی شدند:

- UserRegisterView
- UserLoginView
- UserLogoutView
- Profile View

برای Login و Logout از Class Based Viewهای خود Django استفاده شد.

---

### تعریف مسیرها

در فایل:

```
accounts/urls.py
```

مسیرهای زیر تعریف شدند:

```
/accounts/register/

/accounts/login/

/accounts/logout/

/accounts/profile/
```

همچنین برای جلوگیری از تداخل نام مسیرها از:

```python
app_name = "accounts"
```

استفاده شد.

---

### ساخت خودکار Profile

پس از ایجاد هر کاربر، به وسیله Signal یک Profile نیز به صورت خودکار ساخته می‌شود.

به همین دلیل هیچ کاربری بدون Profile در سیستم وجود نخواهد داشت.

---

# ۴. دستورات ترمینال

اجرای سرور توسعه

```bash
python manage.py runserver
```

ایجاد مدیر سیستم

```bash
python manage.py createsuperuser
```

# ۷. نتیجه‌گیری

در پایان این مرحله سیستم احراز هویت پروژه به صورت کامل پیاده‌سازی شد.

کاربران اکنون می‌توانند:

- ثبت‌نام کنند.
- وارد سیستم شوند.
- از سیستم خارج شوند.
- دارای پروفایل اختصاصی باشند.

این زیرساخت در مراحل بعدی برای پیاده‌سازی امکاناتی مانند علاقه‌مندی‌ها، لیست تماشا، امتیازدهی و ثبت نظر مورد استفاده قرار خواهد گرفت.
