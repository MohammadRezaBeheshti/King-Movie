# پیاده‌سازی سیستم تعامل کاربران

## ۱. هدف مرحله

در این مرحله قابلیت‌های تعاملی کاربران در پروژه King Movie پیاده‌سازی شد.

تا قبل از این مرحله کاربران فقط امکان مشاهده اطلاعات فیلم‌ها و سریال‌ها را داشتند.

با اضافه شدن این بخش، کاربران می‌توانند با محتوای سایت تعامل داشته باشند.

قابلیت‌های اضافه شده:

- علاقه‌مندی‌ها
- لیست تماشا
- امتیازدهی
- ثبت نظر

---

# ۲. معماری سیستم

برای مدیریت تعاملات کاربران یک اپلیکیشن جداگانه ایجاد شد:

```
interactions
```

دلیل جداسازی:

- جلوگیری از شلوغ شدن اپلیکیشن Media
- تفکیک مسئولیت‌ها
- توسعه آسان‌تر قابلیت‌های آینده

---

# ۳. مدل‌های ایجاد شده

## Favorite

ذخیره رسانه‌های مورد علاقه کاربران.

ویژگی‌ها:

- User
- Media
- تاریخ ایجاد

قانون:

هر کاربر نمی‌تواند یک رسانه را چند بار به علاقه‌مندی اضافه کند.

---

## Watchlist

برای ذخیره محتوا جهت مشاهده در آینده استفاده می‌شود.

ویژگی‌ها:

- User
- Media
- تاریخ ایجاد

---

## Rating

سیستم امتیازدهی کاربران.

قوانین:

- هر کاربر فقط یک امتیاز برای هر رسانه دارد.
- بازه امتیاز از ۱ تا ۱۰ است.

---

## Review

سیستم نظرات کاربران.

قابلیت‌ها:

- ثبت نظر
- ویرایش نظر
- حذف نظر توسط صاحب نظر
- مدیریت توسط Admin

---

# ۴. محدودیت‌های دیتابیس

برای جلوگیری از داده‌های تکراری از Constraintهای Django استفاده شد.

مثال:

یک کاربر نمی‌تواند:

- دو بار یک فیلم را Favorite کند.
- چند امتیاز برای یک فیلم ثبت کند.

---

# ۵. مسیرهای ایجاد شده

مسیرهای تعامل:

```
/media/<slug>/favorite/

/media/<slug>/watchlist/

/media/<slug>/rating/

/media/<slug>/review/
```

---

# ۶. امنیت

برای عملیات‌های حساس از:

```python
login_required
```

استفاده شد.

بنابراین:

- کاربران مهمان نمی‌توانند تعامل ثبت کنند.
- هر کاربر فقط اطلاعات مربوط به خودش را مدیریت می‌کند.

---

# ۷. کامپوننت‌های Template

برای جلوگیری از تکرار HTML، بخش‌های رابط کاربری جدا شدند:

```
components/interactions/

favorite-button.html

watchlist-button.html

rating-form.html

review-list.html

review-form.html
```

---

# ۸. اتصال به صفحه جزئیات

کامپوننت‌های تعامل به Media Detail Page اضافه شدند.

اکنون کاربران در صفحه هر فیلم یا سریال می‌توانند:

- محتوا را Favorite کنند.
- به لیست تماشا اضافه کنند.
- امتیاز دهند.
- نظر ارسال کنند.

---

# ۹. پنل مدیریت

برای مدیریت اطلاعات تعاملات، Django Admin توسعه داده شد.

امکانات:

- مشاهده کاربران
- جستجوی نظرات
- فیلتر کردن داده‌ها
- مدیریت محتوا

---

# ۱۰. نتیجه‌گیری

با تکمیل این مرحله، پروژه از یک سیستم نمایش فیلم به یک سامانه تعاملی تبدیل شد.

زیرساخت ایجاد شده در این مرحله پایه قابلیت‌های آینده مانند:

- پیشنهاد فیلم شخصی‌سازی شده
- داشبورد کاربر
- آمار تعاملات
- سیستم رتبه‌بندی پیشرفته

خواهد بود.
