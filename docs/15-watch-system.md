# پیاده‌سازی سیستم پخش (Watch System)

## ۱. هدف مرحله

در این مرحله زیرساخت کامل سیستم پخش پروژه King Movie پیاده‌سازی شد.

هدف این بخش ایجاد بستری برای مدیریت و نمایش منابع پخش و دانلود رسانه‌ها بود، بدون وابستگی به محتوای واقعی.

---

# ۲. مدل MediaSource

برای هر رسانه امکان تعریف چندین منبع پخش یا دانلود فراهم شد.

هر Media می‌تواند چندین MediaSource داشته باشد.

نمونه:

Media

├── 480p Stream
├── 720p Stream
├── 1080p Download
└── 4K Stream

---

# ۳. اطلاعات هر منبع

هر MediaSource شامل اطلاعات زیر است:

- عنوان منبع
- کیفیت
- زبان
- زبان زیرنویس
- نوع منبع (پخش یا دانلود)
- آدرس فایل یا لینک
- حجم فایل (اختیاری)
- وضعیت فعال بودن
- ترتیب نمایش

---

# ۴. صفحه Watch

برای هر رسانه صفحه اختصاصی پخش ایجاد شد.

مسیر:

```
/watch/<slug>/
```

این صفحه شامل موارد زیر است:

- پلیر ویدئو
- اطلاعات رسانه
- کیفیت‌های موجود
- لینک‌های دانلود
- لینک‌های پخش
- رابط کاربری واکنش‌گرا
- پشتیبانی از حالت تاریک و روشن

---

# ۵. مدیریت سریال و انیمه

برای رسانه‌های سریالی:

- فصل‌ها نمایش داده می‌شوند.
- قسمت‌های هر فصل قابل انتخاب هستند.
- هر قسمت صفحه پخش اختصاصی خود را دارد.

فیلم‌ها این بخش را نمایش نمی‌دهند.

---

# ۶. مدیریت منابع

در Django Admin امکان مدیریت کامل MediaSource اضافه شد.

امکانات:

- جستجو
- فیلتر
- مرتب‌سازی
- ویرایش آسان

---

# ۷. ساختار کامپوننت‌ها

برای جلوگیری از تکرار کد، بخش Watch به چند کامپوننت تقسیم شد.

نمونه:

```
components/watch/

player.html
watch-header.html
source-list.html
episode-list.html
```

---

# ۸. فایل‌های محلی و لینک‌های خارجی

سیستم از هر دو روش پشتیبانی می‌کند:

- فایل ویدئویی داخل پروژه
- لینک مستقیم خارجی

این طراحی باعث می‌شود پروژه در آینده بدون تغییر معماری به CDN یا فضای ذخیره‌سازی ابری متصل شود.

---

# ۹. وضعیت فعلی

در نسخه فعلی پروژه از فایل‌های نمونه برای نمایش عملکرد پلیر استفاده می‌شود.

هدف این مرحله پیاده‌سازی زیرساخت سیستم پخش بوده است و تأمین محتوای واقعی خارج از محدوده پروژه است.

---

# ۱۰. نتیجه‌گیری

با تکمیل این مرحله، پروژه دارای زیرساخت کامل برای مدیریت منابع پخش و دانلود شد.

در آینده تنها با اضافه کردن منابع جدید در پنل مدیریت، رسانه‌ها بدون تغییر در کد قابل پخش خواهند بود.
