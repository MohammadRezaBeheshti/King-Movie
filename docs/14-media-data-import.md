# سیستم وارد کردن دیتای رسانه (Media Data Import)

## ۱. هدف مرحله

در این مرحله سیستم ورود داده‌های اولیه برای پروژه King Movie ایجاد شد.

هدف این بخش:

- جلوگیری از ورود دستی اطلاعات از Admin
- آماده‌سازی دیتای واقعی پروژه
- امکان انتقال سریع پروژه به سیستم‌های دیگر

---

# ۲. ساختار داده

اطلاعات رسانه‌ها در قالب JSON نگهداری می‌شوند.

ساختار:

```
data/

└── media_dataset.json
```

این فایل شامل اطلاعات:

- فیلم‌ها
- سریال‌ها
- انیمه‌ها

است.

---

# ۳. Import Command

برای ورود اطلاعات یک Django Management Command ایجاد شد.

مسیر:

```
media_library/

└── management/

    └── commands/

        └── import_media_dataset.py
```

---

# ۴. نحوه اجرا

دستور اجرای Import:

```bash
python manage.py import_media_dataset
```

پس از اجرا اطلاعات JSON خوانده شده و وارد دیتابیس می‌شود.

---

# ۵. مدل‌های پردازش شده

Command اطلاعات را برای مدل‌های زیر ایجاد می‌کند:

## Media

شامل:

- عنوان
- عنوان فارسی
- نوع رسانه
- وضعیت
- توضیحات
- تاریخ انتشار
- امتیاز IMDb

## Genre

ژانرها به صورت خودکار ایجاد می‌شوند.

## Actor

بازیگران در صورت نبود ساخته می‌شوند.

## Director

کارگردان‌ها مدیریت می‌شوند.

## Country

کشورها ایجاد و متصل می‌شوند.

## Season و Episode

برای سریال‌ها و انیمه‌های سریالی:

```
Media

 └── Season

       └── Episode
```

ساخته می‌شود.

---

# ۶. جلوگیری از داده تکراری

Import Command به صورت امن طراحی شده است.

ویژگی‌ها:

- اجرای چندباره باعث ایجاد رکورد تکراری نمی‌شود.
- Media بر اساس slug بررسی می‌شود.
- موجودیت‌های مرتبط دوباره ساخته نمی‌شوند.

---

# ۷. مدیریت خطا

در صورت ناقص بودن داده:

- Command متوقف نمی‌شود.
- فیلدهای اختیاری خالی می‌مانند.
- خطاها در خروجی نمایش داده می‌شوند.

---

# ۸. مزایای این روش

با استفاده از این سیستم:

- راه‌اندازی پروژه روی سیستم جدید سریع‌تر است.
- دیتای پروژه قابل نسخه‌بندی است.
- اضافه کردن محتوای جدید ساده‌تر می‌شود.

---

# ۹. نتیجه‌گیری

پروژه اکنون دارای یک مسیر استاندارد برای ورود داده‌های واقعی است.

به جای وارد کردن دستی محتوا، کافی است فایل JSON بروزرسانی شده و Command اجرا شود.

```
python manage.py import_media_dataset
```
