# معماری Layout پروژه

## هدف

هدف از این مرحله، ایجاد یک ساختار استاندارد برای صفحات وب‌سایت بود تا تمامی صفحات از یک Layout مشترک استفاده کنند و از تکرار کد جلوگیری شود.

---

## ساختار Templateها

```
templates/

├── base.html
├── layouts/
│   └── app.html
├── components/
│   ├── navbar.html
│   ├── footer.html
│   └── theme-toggle.html
└── pages/
```

---

## وظیفه هر فایل

### base.html

این فایل اسکلت اصلی صفحات را در اختیار پروژه قرار می‌دهد و شامل موارد زیر است:

- ساختار HTML
- بارگذاری فایل‌های CSS و JavaScript
- تعریف Blockهای اصلی Template

---

### layouts/app.html

این فایل Layout اصلی سایت را تشکیل می‌دهد و اجزای مشترک صفحات را در خود قرار می‌دهد.

اجزای موجود:

- Navbar
- Main Content
- Footer

تمام صفحات عمومی سایت از این Layout ارث‌بری می‌کنند.

---

### components

کامپوننت‌های مشترک پروژه در این پوشه قرار می‌گیرند.

در این مرحله کامپوننت‌های زیر ایجاد شدند:

- Navbar
- Footer
- Theme Toggle

در ادامه پروژه سایر کامپوننت‌های قابل استفاده مجدد نیز به این بخش اضافه خواهند شد.

---

## مزایای استفاده از Layout

- جلوگیری از تکرار کد
- افزایش قابلیت نگهداری
- جداسازی مسئولیت فایل‌ها
- توسعه آسان صفحات جدید
- امکان ایجاد Layoutهای مختلف برای بخش‌های مختلف سایت

---

## نتیجه

در پایان این مرحله ساختار Templateهای پروژه به صورت ماژولار طراحی شد و زیرساخت لازم برای توسعه صفحات سایت فراهم گردید.
