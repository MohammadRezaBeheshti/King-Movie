# طراحی پایگاه داده

## هدف

در این مرحله، ساختار پایگاه داده پروژه طراحی و پیاده‌سازی شد. از آنجایی که تمام اطلاعات پروژه (فیلم‌ها، کاربران، نظرات و ...) در پایگاه داده ذخیره می‌شوند، طراحی صحیح این بخش اهمیت زیادی دارد. در فریمورک Django، ساختار جداول پایگاه داده توسط کلاس‌هایی به نام Model تعریف می‌شود.

---

## Model چیست؟

در Django هر جدول پایگاه داده توسط یک کلاس Python نمایش داده می‌شود که به آن **Model** گفته می‌شود.

به عنوان مثال، کلاس `Media` معادل جدول `media` در پایگاه داده است و هر ویژگی (Field) این کلاس به یک ستون از جدول تبدیل می‌شود.

نمونه:

```python
class Genre(models.Model):
    name = models.CharField(max_length=100)
```

در این مثال، Django جدولی به نام `Genre` ایجاد می‌کند که دارای ستونی به نام `name` است.

## مراحل ایجاد پایگاه داده

پس از ایجاد Modelها، از دستورات زیر استفاده شد.

### ۱. ایجاد Migration

```bash
python manage.py makemigrations
```

این دستور تغییرات ایجاد شده در Modelها را بررسی کرده و فایل‌هایی به نام Migration تولید می‌کند.

فایل Migration شامل دستورالعمل‌هایی است که مشخص می‌کند چه جدول‌ها و چه ستون‌هایی باید در پایگاه داده ایجاد شوند.

---

### ۲. اعمال Migration

```bash
python manage.py migrate
```

این دستور فایل‌های Migration را اجرا کرده و جداول را به صورت خودکار در پایگاه داده SQLite ایجاد می‌کند.

---

### ۳. ایجاد مدیر سیستم

برای دسترسی به پنل مدیریت Django از دستور زیر استفاده شد:

```bash
python manage.py createsuperuser
```

پس از وارد کردن نام کاربری و رمز عبور، امکان ورود به پنل مدیریت فراهم شد.

---

### ۴. اجرای پروژه

برای اجرای پروژه از دستور زیر استفاده شد:

```bash
python manage.py runserver
```

پس از اجرای پروژه، پنل مدیریت از طریق آدرس زیر در دسترس قرار گرفت:

```
http://127.0.0.1:8000/admin
```
