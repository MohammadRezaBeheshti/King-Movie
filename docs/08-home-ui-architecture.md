# طراحی رابط کاربری صفحه اصلی (Home UI Architecture)

## ۱. هدف مرحله

در این مرحله ساختار رابط کاربری اصلی پروژه King Movie پیاده‌سازی شد.

هدف این مرحله فقط ساخت یک صفحه زیبا نبود، بلکه ایجاد یک معماری قابل توسعه برای رابط کاربری بود تا در ادامه پروژه بتوان کامپوننت‌های مختلف را بدون تکرار کد توسعه داد.

در این مرحله موارد زیر پیاده‌سازی شد:

- طراحی ساختار کامپوننتی Templateها
- ایجاد Layout اصلی
- ساخت Navbar
- ایجاد ساختار نمایش Media
- ایجاد کامپوننت‌های قابل استفاده مجدد
- آماده‌سازی Home Page برای نمایش اطلاعات واقعی از دیتابیس

---

# ۲. معماری Templateها

برای جلوگیری از ایجاد فایل‌های HTML بزرگ و غیرقابل نگهداری، قالب‌ها به بخش‌های کوچک‌تر تقسیم شدند.

ساختار کلی:

```
templates/

├── pages/
│   └── home.html
│
├── components/
│
│   ├── layout/
│   │   ├── navbar.html
│   │   ├── navbar-logo.html
│   │   ├── navbar-links.html
│   │   ├── navbar-actions.html
│   │   └── footer.html
│   │
│   ├── media/
│   │   ├── hero.html
│   │   ├── media-card.html
│   │   └── media-row.html
│   │
│   └── common/
│       └── section-title.html
```

---

# ۳. مفهوم Component در Django Template

در Django Template می‌توان بخش‌های تکراری رابط کاربری را در فایل‌های جداگانه قرار داد.

برای استفاده از یک کامپوننت از دستور زیر استفاده می‌شود:

```django
{% include "components/media/media-card.html" %}
```

مزیت این روش:

- کاهش کدهای تکراری
- تغییر آسان ظاهر کل سایت
- استفاده مجدد از بخش‌ها در صفحات مختلف

---

# ۴. Layout اصلی

تمام صفحات پروژه از یک Layout مشترک استفاده می‌کنند.

ساختار:

```
layouts/app.html
```

این فایل مسئول موارد عمومی سایت است:

- اتصال CSS و JS
- نمایش Navbar
- نمایش Footer
- ایجاد Block اصلی محتوا

صفحات دیگر فقط محتوای خود را داخل Block مربوطه قرار می‌دهند.

---

# ۵. طراحی Navbar

Navbar پروژه به چند بخش تقسیم شد:

```
navbar.html

├── navbar-logo.html
├── navbar-links.html
└── navbar-actions.html
```

وظایف:

## Navbar Logo

نمایش برند King Movie

## Navbar Links

نمایش مسیرهای اصلی:

- خانه
- فیلم‌ها
- سریال‌ها
- انیمه‌ها

## Navbar Actions

شامل:

- جستجو
- وضعیت ورود کاربر
- عملیات مرتبط با حساب کاربری

---

# ۶. طراحی Media Components

برای نمایش فیلم، سریال و انیمه از کامپوننت‌های قابل استفاده مجدد استفاده شد.

## Media Card

مسئول نمایش یک رسانه:

شامل:

- تصویر پوستر
- عنوان فارسی
- سال انتشار
- امتیاز IMDb

این کامپوننت در بخش‌های مختلف استفاده خواهد شد:

- صفحه اصلی
- صفحه جستجو
- علاقه‌مندی‌ها
- لیست تماشا
- صفحات ژانر

---

## Media Row

برای نمایش گروهی از رسانه‌ها استفاده می‌شود.

مثال:

- جدیدترین فیلم‌ها
- جدیدترین سریال‌ها
- محبوب‌ترین‌ها

این کامپوننت یک لیست دریافت کرده و برای هر آیتم Media Card نمایش می‌دهد.

---

# ۷. طراحی سیستم رنگ و Theme

برای مدیریت بهتر ظاهر پروژه، رنگ‌ها به صورت CSS Variable تعریف شدند.

نمونه:

```css
:root {
  --background: #f8fafc;
  --foreground: #111827;
  --primary: #eab308;
}
```

برای حالت تاریک:

```css
.dark {
  --background: #0f172a;
  --foreground: #f8fafc;
}
```

مزیت این روش:

- تغییر آسان تم
- پشتیبانی از Dark Mode
- جلوگیری از تکرار رنگ‌ها در CSS

# ۹. اصول معماری رعایت شده
